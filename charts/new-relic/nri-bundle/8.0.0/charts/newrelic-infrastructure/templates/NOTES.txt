{{- if and .Values.enableWindows (not (include "nriKubernetes.windows.privileged" .)) }}
Note:
=====

Windows nodes are running in unprivileged mode (windows.privileged: false). The Windows kubelet
DaemonSet will use standard container isolation instead of HostProcess containers.

In unprivileged mode:
  - Containers run as ContainerUser
  - Host filesystem access is not available
  - hostNetwork is disabled; the agent uses NRIA_HOST for autodiscovery

For full monitoring capabilities including network statistics, set `windows.privileged: true`
(the default) to run as HostProcess containers with elevated privileges.
{{- end }}

{{- if .Values.kubelet.resources }}
WARNING: `kubelet.resources` behavior is changing
==================================================

Currently, `kubelet.resources` applies the same resource values to all containers in the kubelet
pod (including Windows). New per-container fields are now available and preferred. In a future
release, `kubelet.resources` will be repurposed for pod-level resource setting (once Kubernetes
pod-level resources become generally available), and container-level fallback will be removed.

To silence this warning and gain independent per-container control, migrate to:
  Linux:
    kubelet.kubelet.resources  (nri-kubernetes scraper container)
    kubelet.agent.resources    (infrastructure-bundle agent sidecar)
  Windows:
    kubelet.windows.kubelet.resources  (nri-kubernetes scraper container)
    kubelet.windows.agent.resources    (infrastructure-bundle agent sidecar)

{{- end }}
